# Epic Refund

Браузерная reverse-metroidvania / action-RPG в стиле top-down.
Игрок начинает с 20 навыками и постепенно теряет их у Врат Забвения — это и есть основная прогрессия.

## Быстрый запуск

Проект работает на **чистом JavaScript + Canvas 2D** без сборщиков и внешних библиотек.
Запустить можно через любой статический HTTP-сервер.

### Python (встроен в большинство систем)

```bash
cd epic-refund
python3 -m http.server 8000
```

Открой `http://localhost:8000` в браузере.

### Node.js (если есть)

```bash
cd epic-refund
npx http-server -p 8000
```

### VS Code

Расширение **Live Server** → правый клик по `index.html` → *Open with Live Server*.

## Debug-режим

Добавь `?debug` к URL: `http://localhost:8000?debug`

В консоли браузера доступен объект `window.game.debug`:

```javascript
window.game.debug.getSkillTree()           // Текущее дерево навыков
window.game.debug.removeSkill('str_t4')    // Убрать конкретный навык
window.game.debug.getEchoes()              // Активные эхо-бонусы
```

## Управление

| Клавиша | Действие |
|---|---|
| `WASD` / стрелки | Движение (8 направлений, диагонали нормализуются) |
| `J` | Лёгкая атака (комбо до 3 ударов) |
| `K` | Тяжёлый удар / парирование (если есть навыки) |
| `Shift` | Перекат в направлении движения |
| `E` | Взаимодействие с NPC / Вратами |
| `U` `I` `O` `P` | Слоты навыков 1–3 / ультима |
| `Esc` | Закрыть меню Врат |

## Как играть

1. Стартуешь в **Столовой (room_01)** рядом с торговцем Гриблом. Нажми `E` рядом с ним — услышишь его мнение о тебе.
2. Иди на восток. Чтобы перейти в следующую комнату — нужно **зачистить текущую** (все враги должны быть повержены), после чего двери откроются сами.
3. В комнате 2-4 встретишь **Врата Забвения** — они не пропустят, пока не пожертвуешь навыком. Подойди и нажми `E`, выбери навык стрелками, подтверди `J` или `E`.
4. После потери навыка ты станешь чуть слабее, но получишь **Эхо** — слабый пассивный бонус от утраченного навыка.
5. Дойди до комнаты 2-7 — там конец уровня.

## Добавление ассетов (спрайты и звуки)

Весь код уже работает **без единого ассета** — используются плейсхолдеры (прямоугольники с подписями).
Замена на спрайты — одна строка в `src/main.js`:

```javascript
// После game.init(canvas), до game.loadScene():
game.spriteRegistry.register('player', 'assets/sprites/player.png', { width: 14, height: 14 });
game.spriteRegistry.register('slime',  'assets/sprites/slime.png',  { width: 16, height: 12 });
game.spriteRegistry.register('goblin', 'assets/sprites/goblin.png', { width: 14, height: 16 });
```

Форматы:
- Спрайты — PNG/GIF/WebP, любой размер
- Звуки — MP3/OGG/WAV через `game.audioManager.loadSFX('swing_light', 'assets/audio/swing.wav')`

Пока звуков нет — проигрываются **процедурно сгенерированные** через WebAudio API (меч, удар, парирование, потеря навыка и т.д.).

## Архитектура

```
src/
├── core/              EventBus, Game, InputManager, Camera, Renderer, SaveManager, AudioManager
├── systems/           SkillTree, EchoSystem, CombatSystem, FearSystem, HUDManager, GateSystem, WeightSystem, ParticleSystem
├── entities/          Player, Enemy, NPC + конкретные враги и NPC
├── levels/            Tilemap, Room, Level, Level02_Barracks
├── render/            SpriteRegistry, WorldRenderer
├── ui/                HUD, GameScene, DialogueSystem, GateUI, Notifications
├── data/              balance.js, skills.js, dialogues/kane_dialogues.js
└── utils/             MathUtils, ObjectPool
```

**Правила кода:**
- Системы общаются только через `EventBus` — никаких прямых зависимостей
- Все числа — в `src/data/balance.js`, никакого хардкода в логике
- Данные навыков — в `src/data/skills.js`

## Ключевая механика: обратное дерево навыков

Игрок начинает с **20 навыками** в 5 ветках (Сила, Магия, Защита, Ловкость, Аура).
Каждая ветка: 4 тира, от базы к ультимативному навыку.

**Правило обрезки:** удалить навык можно только если у него нет активных зависимых.
То есть по ветке — только **с конца к началу**.

Потерянный навык оставляет **Эхо** — слабый пассивный бонус.

## Фазы игрока

- `god`      — 17–20 навыков (золотистый цвет)
- `hero`     — 12–16 (серебряный)
- `veteran`  — 7–11 (бронзовый, реплики иронические)
- `survivor` — 3–6 (тусклый)
- `nobody`   — 0–2 (серый, почти не говорит)

Цвет плейсхолдера игрока меняется вместе с фазой.
